Hero 汇总

2023-04-26 备忘 Hero, 备忘, 服务 0 评论 字数统计: 1.9k(字) 阅读时长: 9(分)

记录在使用Hexo过程中的一些心得体会。

在挑选了一些模板测试后发现功能都太多太复杂了，不符合我建立博客的初衷。然后就有了现在使用的模板pure，但是还有些不满意的地方。

  1. 评论功能。这个博客主要是为了记录生活工作中的一些心得和一些防止我忘记的东西，不需要评论也没有精力去管理这些评论。
  2. 摘要功能。主页的文章是有摘要的，但是分类页和标签页没有。以后有可能要写些卡片笔记放在博客里，没有摘要会很不方便。
  3. 代码或内容折叠。最好是内容折叠，这样页面就不会太长影响我快速浏览整个页面。
  4. 历史功能。 页面修改后不知到之前的内容。
  5. 文档加密。
  6. 免密部署。

还有些待解决的问题放在下面，慢慢研究有了解决方案在补上：

从theme-next/hexo-theme-next扒tags没有成功 已经拷贝了scripts/tags和source/tags的相关代码和修改了css的导入 还是没有成功，看来要系统的学习下主题的制作才能解决这个问题了。也许hexo可以出个系列？

关闭评论功能

评论功能是通过_config.pure.yml配置文件中的comment 字段来配置的。但是不能关闭，只能选择不同的服务商。研究了会配置也没找到好的方法。按照我的习惯找不到配置项就直接删代码。没有评论功能的代码他总是能取消这个功能了。通过搜索主题目录在静态页上的一些关键字找到了相关功能在\pure\layout_partial\post\comment.ejs里面。通过这个文件把评论框插入到了页面上。通过清空这个文件顺利删除了所有页面的评论功能。

历史功能

历史功能暂时没有想到特别好的方法。现在的解决方案是打开gogs然后将内容同步到上面，虽然不需要通过同步来部署内容但是可以通过这个方法保留历史记录。

文档加密

安装hexo-blog-encrypt插件 配置_config.yml文件添加相关配置。然后hexo s 的时候输入密码回车没反应 一直以为是主题的问题。弄了一会没弄好通过F11 控制台发现是没有打开ssl 赶紧在HS上配置nginx然后再试发现一切正常。

github免密部署

修改_config.yml 的 repo 使用ssh地址 本地用户名和github上的一致 sshkey已经正常保存至GitHub上。
摘要 设置 SSH 使用 hexo deploy 时免输用户名密码
